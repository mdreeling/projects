Concept - TrackMyGear

TrackMyGear is an application for creating an inventory of musical studio equipment such as Guitars and Microphones ( tems which frequently go walkabout during and after recording sessions)

With TrackMyGear you can categorise and take photographs of the gear itself as well as its serial number and upload it to the TrackMyGear site.

The application is hosted on Amazon EC2 @ http://www.trackmygearfor.me

UI Inspiration - TrackMyGear

TrackMyGear uses the jquery standard library features but does not use jquery-mobile.

As part of a learning exercise i tried to move out of jquery-mobile and into standard css.

I based all of my UI work on the following blog post which tries to emulate an iOS style interface.

http://cheeaun.com/blog/2012/03/how-i-built-hacker-news-mobile-web-app

Feature and Technical Notes

  1. PostMark integration only operates in the Mobile Web version - but operates on all devices in this mode
  2. All images are stored as base64 encoded text in MongoDb
  3. Twitter Integration on the iPhone is handled by a native plugin. You need to have your twitter account setup on the Phone. I had to submit an Objective-C patch to the Twitter plugin. I added a getTwitterUsername call in order to make it usable for my application. This patch has since been merged to the TwitterPlugin master as it is quite useful.
  4. I also submitted a pull request for the Flurry plugin because it does not work with the latest version of Cordova

Known issues

  1. Android - Native and Web - Scrolling only works on Ice Cream Sandwich (4.0.1) upwards
  2. iOS - Native - Scrolling is sporadic - hence videos to show that it works - i think this is due to webkit-overflow
  3. Edit item - All Versions - Not implemented fully
  4. Sometimes the Node.JS server on trackmygearfor.me will crash when checking the twitter promise. Webmin is configured to restart the Node.JS server after 60 seconds when this occurs. You can try again after this period.
