USC03 - Add Gear (Native and Mobile Web)

The user adds some gear using TrackMyGear

Actors:
User

The user attempting to use TrackMyGear

TrackMyGear

The TrackMyGear inventory management system

 

Triggers:
1

The user presses the "+" button on the top left of the application

Preconditions:
1

The user has browsed to http://www.trackmygearfor.me and is logged in

Post-conditions:
1

The user is redirected to http://www.trackmygearfor.me#list

2

The users inventory is displayed as a list with images including the new item

User Flows

The standard flow describes what happens if the user is successfully logged into twitter and hence TrackMyGear

Standard Flow

1. The user clicks the "+" button
2. The Add Gear div slides in
3. The user adds Make, Model Serial Number and Category and Description (Mobile Web)
4. The user also adds a Serial Number and Equipment Image (Native)
5. The user presses save
6. The user is redirected to the #list div
7. The list is displayed including the newly added item

Alternate Flows
1A: (Native) No image selected

1. At step 4 of the normal flow the user does not take a photograph
2. An error message is displayed when the user returns to the application
3. The users is returned to step 2 - however their entered data is still present
